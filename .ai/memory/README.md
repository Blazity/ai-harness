# AI Memory

Persistent product knowledge for AI Harness. Read this directory for durable context about what the project is, how it is built, and which implementation constraints matter.

| File | Purpose | Update when |
| --- | --- | --- |
| [`product.md`](product.md) | Product purpose, audience, and scope | Product direction changes |
| [`architecture.md`](architecture.md) | Technical invariants and module boundaries | Architecture or CLI behavior changes |
| [`stack.md`](stack.md) | Runtime, tooling, dependencies, and constraints | Tooling or dependency choices change |
| [`lessons.md`](lessons.md) | Append-only pitfalls and learnings | A non-obvious lesson is discovered |

Vocabulary lives in [`../LANGUAGE.md`](../LANGUAGE.md). Implementation plans live in [`../plans/`](../plans/). Research lives in [`../research/`](../research/). Decisions and ADRs live under the configured decisions paths in `.ai/config.json`.
